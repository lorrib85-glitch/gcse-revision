Question bank files are organised by subject (`history/`, `science/biology/`, `maths/`, `english/`). Each file exports a `BY_TOPIC` object keyed by topicId (for backward compat with `QUESTION_BANKS_BY_MODULE`), an optional `QUICKFIRE` array for MC-format quickfire questions, and a flat `ALL_*_QUESTIONS` array. The `questionRegistry.js` file at the root of this folder re-exports everything and provides utility query functions. The `questionTypes.js` file exports FORMAT, DIFFICULTY, and EXAM_TYPE constants shared across all banks.

Question `tags` use the canonical learning graph vocabulary (`src/data/learningGraph/`, see `docs/system/LEARNING_GRAPH.md`): facet tags (`period:medieval`, `skill:recall`, `exam-type:explain-why`) plus registered concept ids (`history:medicine:galen`). Never invent a new concept spelling in a bank — register it in the concept registry first; `tests/architecture/learning-graph.test.js` fails on unregistered subject-namespace tags.
